# Skill: Presentations (PPTX & HTML Slides)

## When to use
Any request for slides, presentations, decks, pitch decks, reports, or visual storytelling.

## Two Output Formats

### HTML Slides (preferred for speed and style)
- Generate a self-contained HTML file with CSS animations
- Use reveal.js or a clean custom CSS grid layout
- Save to /tmp/presentation.html and send as file
- Fully interactive in any browser, no software needed

### PPTX (for Office compatibility)
- Use python-pptx library to generate
- Save to /tmp/presentation.pptx and send as file

## Slide Design Principles
- 1 idea per slide — never wall-of-text
- Strong headline that states the insight (not topic)
- Max 3-5 bullet points per slide, each under 10 words
- Data slides: one chart, one insight headline
- Use consistent dark or light theme throughout

## Oui's Presentation Context
- Internal Agoda presentations: data-driven, executive-ready
- Conference/external talks: thought leadership, engaging
- AI/automation demos: visual, show-don't-tell
- Always include: title slide, agenda, key sections, conclusion/next steps

## Workflow
1. Ask for (or infer): topic, audience, number of slides, key messages
2. Draft outline first if topic is complex
3. Generate full presentation
4. Save file to /tmp/ and send via Telegram

## Structure Templates

### Executive Report (5-8 slides)
1. Title + date
2. Executive Summary (3 bullets)
3. Current State / Problem
4. Solution / Approach
5. Results / Impact (metrics)
6. Next Steps

### Conference Talk (10-15 slides)
1. Title + speaker
2. Who am I / context
3. The problem/opportunity
4. 3-4 content sections
5. Demo or case study
6. Key takeaways
7. Q&A / Thank you

## Examples
- "Create a 10-slide deck on AI automation ROI for FinTech"
- "Make an HTML presentation for my team about data pipeline improvements"
- "Build a pitch deck for the new AI initiative at Agoda"
