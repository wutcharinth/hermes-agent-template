# Skill: HTML & Web Creation

## When to use
Landing pages, dashboards, interactive tools, data visualizations, web apps, portfolio pages, any browser-based output.

## Output Method
- Generate complete self-contained HTML file (all CSS/JS inline or CDN)
- Save to /tmp/output.html
- Send as file via Telegram (opens in any browser)

## Tech Stack Preferences
- **Styling:** Tailwind CSS (CDN) for fast professional UI
- **Charts:** Chart.js for data visualization
- **Interactivity:** Vanilla JS or Alpine.js (no heavy frameworks)
- **Icons:** Lucide or Heroicons via CDN
- **Fonts:** Google Fonts (Inter, Plus Jakarta Sans)

## Design Standards
- Mobile-responsive by default
- Dark mode preferred for dashboards and tools
- Light/clean for landing pages and reports
- Always looks professional — not like a prototype

## Common Use Cases

### Dashboard / Report
- KPI cards at top
- Charts in grid layout
- Data tables with sorting
- Dark theme, professional colors

### Landing Page
- Hero section with headline + CTA
- Features/benefits section
- Clean, modern, conversion-focused

### Interactive Tool
- Form inputs with real-time calculation/output
- Clean UX, instant feedback
- No backend needed — pure client-side

### Data Visualization
- Chart.js for bar, line, pie, scatter
- Always label axes and include legend
- Color-accessible palette

## Oui's Context
- May need dashboards for internal Agoda data presentations
- FinTech KPI dashboards, pipeline monitoring views
- Demo pages for AI/automation capabilities

## Examples
- "Build an HTML dashboard showing these metrics: [data]"
- "Create a landing page for the AI Automation team at Agoda"
